=== Chapter 00 ===

Chapter 0 · How to read this book

Learning objectives

  Name the five visible stack layers bottom-up and what each owns.

  Distinguish cold boot (systemd/KILROY kernel) from warm boot (field stack scripts).

  Open the correct loopback URL for desktop vs browser without swapping them.

  Run three health checks that prove the stack is alive.

  Locate the doctrine JSON for any component you are debugging.

Who this book is for

  Operators bringing up SG/NewLatest on a workstation, QEMU smoke box, or field
  laptop. Hostess 7 teaches this corpus into brain storage; you read it here in
  plain text so grep and library shelves work offline.

  If you already read Field Technology v5, treat this book as the map of cities.
  v5 is the geology under one city (AMOURANTHRTX fabric). Exploring the Stack
  is the transit diagram — which train stops where, which ticket opens which door.

Honesty labels (same as Field Technology)

  Implemented — file path or API returns JSON today.
  Wired — script calls module; may need build step first.
  Doctrine — JSON says so; verify before demo.
  Retired — historical; do not wire new work here.

Plain English

  The stack is a set of programs that agree on 127.0.0.1 ports and env vars.
  AmmoOS is your fullscreen classic desktop (Start button, taskbar). Queen is
  the secured browser you launch from that desktop. They are siblings — Queen
  does not contain AmmoOS and AmmoOS does not boot inside Queen tabs.

  here — not a separate gap tier.

Chapter 11 · Boot process — complete walkthrough

11.1 Operator paths (pick one)

  Path A — Fast dev (most common on SG workstation)
    cd NewLatest
    ./lib/ammolang-run.sh stack_fast
    OR: bash scripts/impl/ammoos-direct-start.sh

    bash scripts/impl/start-field-stack.sh
    OR: ./scripts/stack.sh start

    bash scripts/impl/kilroy-load-os.sh

  Path D — QEMU smoke (no desktop UI)
    ./lib/ammolang-run.sh boot_qemu_test

11.2 ammoos-direct-start.sh sequence (wired)

  1. Export sibling env (AMMOOS_INSIDE_QUEEN=0, QUEEN_STANDALONE_BROWSER=1, …)
  2. ZNetwork orchestrator mark-running if operator.json missing
  3. field-dns.sh + nexus_field_services_boot
  4. panel-json-assemble.py if threat-panel.json missing
  5. threat-panel-http.py :9477 (panel static root)
  6. queen-world.py --daemon :9481
  7. Poll until :9477/field and :9481/api/status return 200

  Logs: .nexus-state/panel-http.log · queen-world.log

11.3 start-field-stack.sh sequence (full)

  1. nexus_init_runtime_paths + security tier (extreme vs consumer)
  2. kilroy-core.sh board (KILROY PC core markers)
  3. nexus-vestigial-cleanup.sh
  4. integrate-znetwork.sh → source znetwork-integrated.env
  5. znetwork-field.sh startup
  6. AmmoCode :9555 if down
  7. Panel + Queen + Final_Eye + defenses (continues below line 155+)
  8. Hostess7 bridges, sense meld, ironclad seeds per manifest

  Env: NEXUS_BOOT_C2_ONLY=1 boots desktop-first, not Queen auto-launch.

Chapter 12 · Health checks and troubleshooting

12.1 Three-line stack proof

  curl -sf -o /dev/null -w '%{http_code}\n' http://127.0.0.1:9477/field
  curl -sf -o /dev/null -w '%{http_code}\n' http://127.0.0.1:9481/api/status
  ss -tlnp | grep -E ':9477|:9481'

  Expect: 200 · 200 · listeners present

12.2 Desktop posture

  pythong lib/field-host-desktop.py build | jq '.shell.boot_program,.shell.display_backend'

12.3 Queen browser posture

  pythong Queen/lib/queen-browser.py json | jq '.stripped,.boot_os,.start_button,.home'

12.4 KILROY + ZNetwork

  curl -sf http://127.0.0.1:9481/api/kilroy | jq '.kilroy_present,.rank'
  curl -sf http://127.0.0.1:9477/api/znetwork | jq '.ok,.mode'

12.5 AMOURANTHRTX canvas

  pythong Queen/lib/queen-canvas-renderer.py json | jq '.amouranthrtx_present,.ok'

12.6 Common mistakes

  Symptom: Queen shows KILROY home but no desktop
  Cause: You opened Queen browser expecting OS — open :9477/field instead.

  Symptom: MONSTER popup during stack_start
  Cause: AmmoLang task stalled (grok_lab_boot chain). Wait or force quit tree.

  Symptom: :9477 connection refused
  Cause: Legacy NEXUS-Shield on :9477 or panel not started. Run ammoos-direct-start.

  Symptom: Queen and terminal identical screenshots
  Cause: URL smoke test only — not desktop click-through. Launch Queen from Start.

  Symptom: RTX binary ignored
  Cause: QUEEN_SKIP_RTX_BOOT=1 default. Build g16-build.sh then unset or auto-detect.
